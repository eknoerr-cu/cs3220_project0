## User Story Example 1 (Must)

**User Story:**

> As a student, I want to add courses to my semester schedule so that I can plan my workload.

**Acceptance Criteria:**

* Given the user is logged in, when they select “Add Course,” then a course entry form is displayed.
* The form allows entry of course name, course code, and number of credits.
* When required fields are missing, the system displays a validation message.
* After submission, the course appears in the user’s semester schedule list.
* The course persists after page refresh.

## User Story Example 2 (Should)

**User Story:**

> As a student, I want to view my weekly schedule so that I can see time conflicts.

**Acceptance Criteria:**

* The system displays courses in a week-based calendar view.
* Courses with overlapping times are visually distinguishable.
* The calendar updates automatically when a course is added or removed.
* If no courses exist, the system displays a helpful empty-state message.
